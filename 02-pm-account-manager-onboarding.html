/06 — MONTHLY REPORTING

The report is the product.

If clients only see one deliverable per month, it's this one. Treat it like a finished piece of writing.

Non-negotiables

01
Always use the template. monthly-report-template.md — don't improvise structure.
Rule
02
Numbers first, narrative second. Lead with the KPIs the client cares about. Then explain why.
Rule
03
At least one recommendation. Log it in recommendations-tracker.md. Every. Single. Month.
Rule
04
Consent-impact always mentioned. 34% sessions hidden by consent — remind them what we're recovering.
Rule
05
On brand. Run the marketing:brand-review skill on the copy before it goes out.
Rule

Report anatomy

# Monthly Report · [Client] · [Month] ## Executive Summary - 3 bullets. Outcome-first. ## Performance - ROAS / CPA / Conversions vs last period - Channel breakdown ## Tracking Health - Consent mode coverage - Conversions hidden vs recovered - Anomalies detected ## Recommendations - Rec #1 ← logged in tracker - Rec #2 ← logged in tracker ## Next Month - What we'll ship - What we'll need from you
/07 — CLIENT HEALTH SCORING

Score every client. Every month.

We don't wait for churn signals — we predict them. The health-scoring model (in 03-Resources/Playbooks/client-health-scoring-model.md) runs automatically. Below 70 = alert.

90+
Thriving — pitch expansion
70–89
Healthy — stay consistent
50–69
At risk — intervention
<50
Critical — save plan active

What drives the score

Engagement

Response time to emails. Attendance on calls. Whether they read monthly reports.

Delivery

SLA on tickets, tracking uptime, time since last meaningful deliverable.

Financial

Invoice payment time. MRR direction. Tier maturity.

Sentiment

Extracted from ## Client Signals. Objections, praise, feature requests.

Data quality

Are we still proving value? Is ROAS trending up or flat?

Strategic fit

Do they still match our ICP? Is the use case still compelling for them?

/08 — THE LEARNING LOOP

Close. Every. Loop.

This is what separates Amplio from every other agency. Recommendations don't float — they get logged, measured, and fed back into the pattern library. Three behaviors, memorize them.

Behavior 01

Log recommendations

Said something to a client — in a meeting, email, or report? Append it to their recommendations-tracker.md. Priority, status, linked pattern ID.

# Recommendation 2026-04-13 Client: Odisei What: Move Meta campaigns to server-side conversion API Priority: High Status: Proposed Pattern: [[P-037-sCAPI-lift]]
Behavior 02

Extract signals

Any client interaction — call, email, Slack — gets processed. Feature requests, objections, praise, satisfaction all land in the client's ## Client Signals section in _overview.md.

## Client Signals 2026-04-10: (Objection) "We don't want another dashboard" 2026-04-12: (Request) Wants weekly ROAS snapshots 2026-04-13: (Praise) "The consent-mode fix was huge"
Behavior 03

Test patterns

When you review data, actively check if it confirms or contradicts existing patterns in cross-client-patterns.md. Three contradictions in 30 days downgrades a pattern's confidence.

# Pattern test log P-037: sCAPI lifts ROAS 15-25% Certify — confirmed (+22%) Odisei — confirmed (+18%) Birdie — contradicted (+3%)
/09 — WEEKLY & MONTHLY RITUALS

The rhythm that keeps it all working.

Show up consistently. Nothing heroic. The system is designed to make a disciplined AM look like a miracle worker.

Every morning — Session brief+
"Session brief" in Claude. Read the state of the world. Glance at pipeline. Review any alerts from client health scoring.
Every afternoon — Process inbox+
"Process inbox" — Brain-Dump.md gets routed. Any "update_" labeled Gmail threads get synced to the Vault.
Every Friday — Client health+
"Client health" — run the full scoring pass. Any client <70 gets a save plan drafted before Monday.
Every 1st of month — Monthly reports+
One report per client per month. Use the template. Run brand-review on the copy. Send with a calendar invite for the review call.
Quarterly — Pattern review+
"Test patterns" + "Learning report". Cross-client patterns get confirmed, downgraded, or retired. Playbooks get updated.
Always — Update memory+
End every work block with "Update memory". CURRENT STATE gets overwritten, log gets appended. Future-you thanks present-you.
