/04 — VAULT ARCHITECTURE

The Vault.
One brain. Three layers.

A version-controlled Second Brain for the whole company. Built on the Karpathy LLM-wiki pattern — raw sources compile into a structured wiki, governed by a schema. Claude reads it, writes to it, and keeps it alive.

Pattern
Karpathy three-layer LLM wiki
Structure
IPARAG + Client Spaces
Schema
CLAUDE.md & program.md
Agent
Claude, bidirectional commands
/01 — THE STACK

Raw → Wiki → Schema.

Three layers, each with a distinct job. Raw is immutable ground truth. The Wiki is compiled, structured, interlinked. The Schema is the rule-book that governs how both behave — which is what you're reading right now.

Layer01

Raw Sources

00-Inbox/raw/

Articles, PDFs, transcripts, web clips, brain-dumps. Immutable after capture — never edited. The ground truth every compiled note is traceable back to.

QualityImmutable · Cited
↓
Layer02

The Wiki

05-Galaxy/ · 03-Resources/

Atomic Galaxy notes — one idea, ≤300 words, one-line summary, ≥1 wikilink. Plus Resources: playbooks, templates, brand, skills. This is where work gets done.

QualityCompiled · Interlinked
↓
Layer03

The Schema

CLAUDE.md · program.md

The rules. Routing, behaviour, priorities, commands, priorities for the autoresearch loop. Every session reads this first, so the Vault behaves consistently.

QualityAuthoritative · Versioned
Raw sources get compiled into structured Galaxy notes and Resources. The Schema governs how the compile happens. Claude is the runtime. — The operating model in one sentence

/03 — ROUTING

Every input has one home.

The routing table is unambiguous. If it's a raw source, it's immutable. If it's a client thing, it lives in their space. If it's a deadline, it's a project. If it's a concept, it's Galaxy.

/05 — AUTORESEARCH

Compile. Lint. Enrich.
On repeat.

The Vault is a live system. Raw sources get compiled into Galaxy notes. Broken links and stale content get linted. Thin notes get enriched with fresh research. A health score decides when the loop needs to run again.

/ Step 01

Capture

Anything of value lands in 00-Inbox/raw/. Frontmatter gets processed: false.

/ Step 02

Compile

Extract concepts, create/update Galaxy notes, cross-link to related. Mark raw as processed: true.

/ Step 03

Lint

Broken wikilinks, 404 external URLs, missing summaries, stale updated: dates.

/ Step 04

Enrich

Thin notes get research passes. Fresh facts. Evidence added. Tagged ai-enriched.

/ Step 05

Score

Vault health score. Below threshold → the improvement loop schedules another pass.

Schedule

Daily · Background

Inbox sweep

Light routing pass. Brain-Dump items get homed. No writing to Galaxy yet.

Weekly · Compile

Raw → Galaxy

Every unprocessed raw source gets compiled. New atomic notes, new wikilinks, cross-references.

Weekly · Lint

Broken-link sweep

Checks wikilink integrity, validates external URLs, flags notes without summaries or Related sections.

Monthly · Enrich

Research pass

Thin notes and notes near a priority topic (see program.md) get fresh research.

Per session · Score

Health score

Quality metric. Below threshold triggers the improvement loop the next run.

/06 — HUB GRAPH

Seven hubs.
No orphans.

Every note in the Vault links to at least one hub. Hubs are the spine — indexes of indexes. This is what makes the wikilink graph navigable instead of a pile of files.

Solid lines

Every hub links back to [[memory]] — session continuity first.

Dashed lines

Cross-hub links. Galaxy ↔ Skills, Clients ↔ Pipeline, etc.

Satellite dots

Leaf notes — every one links to at least one hub via its Related section.

/07 — PROVENANCE

Know where it came from.

Every note carries its origin. Human-authored content and AI-enriched content are clearly distinguishable in the frontmatter — so we can weight trust, audit sources, and measure the autoresearch loop over time.

/ Human-authored

tags: [sales, positioning]

No ai-enriched tag. No compiled_from: frontmatter key. Voice is yours. Claim-level decisions are yours.

/ AI-enriched

tags: [analytics, consent, ai-enriched]

ai-enriched in tags. If compiled from a raw source, also compiled_from: [[raw/filename]]. Traceable end-to-end.

If we can't trace it back to a human or a cited raw source, we don't claim it. — Verified metrics only · Rule 03
/08 — BIDIRECTIONAL COMMANDS

Say it.
Claude does it.

Fixed phrases that trigger defined behaviour — creating folders, routing files, updating indexes, scoring leads, surfacing signals. The Vault is driven by commands, not ad-hoc requests.

CommandWhat happens
"New client: [name]"Creates 06-Clients/[name]/ from template. Updates _Client-Index.md. Runs onboarding intelligence pre-loader.
"New prospect: [name]"Creates record in 06-Clients/_Prospects/. Updates Pipeline Dashboard.
"Graduate [prospect]"Moves prospect to client space, expands to full template.
"Process inbox"Reads Brain-Dump, routes each item to its home, clears processed entries.
"Compile the vault"Full compile pass. Reads unprocessed raw, extracts concepts, creates/updates Galaxy notes.
"Lint the vault"Deep lint. Broken links, stale content, missing summaries, orphan notes.
"Client brief: [name]"Reads _overview.md + recent notes. Returns status summary.
"Meeting notes: [client]"Creates dated note in client's meeting-notes/ folder.
"Score this lead: [company]"Reads ICP file, returns score + tier + rationale.
"Track recommendation: [client]"Adds row to client's recommendations-tracker.md.
"Client health"Runs health scoring across active clients. Table with scores, trends, alerts.
"Test patterns"Contradiction-testing pass. Confirms or downgrades pattern confidence.
"Learning report"Summarises actioned recommendations, pattern movements, signal themes, health changes.
"Update memory"Overwrites CURRENT STATE in memory.md. Appends to rolling log.
/09 — THE THIRTEEN

The thirteen
non-negotiables.

Governance on one page. Every behaviour rule is codified in CLAUDE.md. Claude reads them before every session. Contributors commit to them.

/ 01

Load on demand

Don't recite brand voice or ICP from memory. Read the file.

/ 02

Vault-first

Search the Vault before giving advice. Knowledge may already exist.

/ 03

Verified metrics only

Only the proof points from CLAUDE.md. Never fabricate.

/ 04

Cross-link always

Client notes link to Galaxy. Galaxy links back.

/ 05

Keep the Vault alive

Every session should leave it richer than it started.

/ 06

Ask before assuming

If a command is ambiguous, clarify before moving files.

/ 07

No orphans

Every new note has a Related section with ≥1 hub wikilink.

/ 08

update_ emails → sync

Amplio Updates emails update client _overview.md, pipeline, index.

/ 09

Close the loop

Every recommendation to a client goes in their recommendations-tracker.md.

/ 10

Extract signals

Every client comms: feature requests, objections, churn indicators → overview.

/ 11

Test patterns

Actively look for contradictions. Three in 30 days downgrades confidence.

/ 12

Every client gets a health score

Below 70 triggers alerts. No active client is unmeasured.

/ 13

No data-context gaps

Every active client has an API or manual data-context.md.

The Vault is the operating system. The thirteen rules are the kernel. Claude is the shell. — Architecture, distilled
