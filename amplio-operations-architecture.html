System Design

Architecture Overview

Amplio operates across three integrated pillars. Each layer feeds the next, creating a closed-loop system of knowledge, data, and action.

Knowledge Vault

IPARAG folder structure with three-layer architecture (raw sources → compiled wiki → schema rules). Session continuity via memory.md.

Knowledge Architecture

Three-Layer Architecture (Karpathy LLM Wiki)

Inspired by Andrej Karpathy's approach to building a searchable wiki, Amplio uses a three-layer compilation model.

Layer 1: Raw Sources (00-Inbox/raw/)
Immutable inputs — articles, PDFs, transcripts, web clips. Never modified. Marked processed once compiled.
Layer 2: Wiki (05-Galaxy/ + 03-Resources/)
LLM-compiled structured knowledge. Atomic notes (≤300 words) with wikilinks. One idea per note.
Layer 3: Schema (CLAUDE.md + program.md)
Rules governing compilation. Commands, routing logic, behavior rules, hub connections.

Continuous Improvement

Vault Autoresearch Loop

The core autonomous improvement cycle. Runs daily at 09:00 AM, makes one atomic change per iteration.

Step 1
Score the Vault

Compute six-dimension health score (structural integrity, hub connectivity, frontmatter, links, freshness, client data).

Step 2
Identify Highest-Impact Improvement

Rank pending improvements by dimension, priority tier (P0 → P1 → P2 → P3), and expected impact.

Step 3
Make One Atomic Change

Add link, fix broken reference, create missing note, update stale content. One change only.

Step 4
Re-Score

Measure change impact. If score improved, commit. If not, revert.

Step 5
Log Result

Append to memory.md log. Track impact, dimension, and reasoning.

Priority Tiers:

P0 Structural Integrity — broken links, invalid YAML

P1 Hub Connectivity — notes without hub links

P2 Content Quality — missing summaries, stale notes

P3 Client Data Hygiene — incomplete overviews, missing sections

Knowledge Extraction

Compile Operation

Automatic transformation of raw sources → Galaxy notes. Runs on-demand or as part of autoresearch. Immutable sources never modified.

Input: Read Raw Sources
Scan 00-Inbox/raw/ for unprocessed files. Immutable — never edits original.
Extract Concepts
Identify key ideas, entities, relationships. One atomic concept per note.
Create Galaxy Notes
Write atomic notes (≤300 words) with YAML frontmatter, summary line, body, wikilinks.
Cross-Link
Connect to existing vault content, hub indexes, client spaces, patterns.
Mark Raw Source Processed
Add frontmatter: processed: true, compiled_to: [[note-name]]. Body never modified.

Provenance Rule: AI-generated or AI-enriched content gets ai-enriched in tags. Notes compiled from raw get compiled_from: [[raw/filename]] in frontmatter.

Feedback Systems

Learning Layer

The closed-loop feedback system. Tracks what Amplio recommends, what clients do, and what results.

Recommendation Tracking

Every recommendation → recommendations-tracker.md. Captures: text, pattern ID, priority, outcome, date.

Client Signal Extraction

Qualitative signals from comms. Categories: Feature Request (FR), Objection (OB), Satisfaction (SAT), Churn Risk (PS), Exit Intent (EX).

Health Scoring

0-100 composite per client. Signals feed score. Scores <70 trigger alerts. Recalculated after each client touchpoint.

Pattern Contradiction Testing

When new client data contradicts an existing pattern, log it. Three contradictions in 30 days → auto-downgrade confidence level (High → Medium → Low). High-confidence patterns become playbooks.

Example: Pattern A-102 (Google Ads bid-up improves ROAS) contradicted by Certify Education data. Two other contradictions exist. Total: 3 in 28 days → downgrade from High to Medium.

Experimental Automation

Ad Campaign Autoresearch

Applied to Certify Education Google Ads. Four experiment types (E0–E3) with guardrails and auto-execution thresholds.

Experiment Types (Escalation Model)

Type Description Execution
E0 Bid/Budget adjustments (±20% max) Auto-execute if score ≥70
E1 Targeting refinement (keywords, audiences) Auto-execute if score ≥70
E2 Copy optimization (headlines, descriptions) Always flagged for review
E3 Campaign structure changes (consolidation, splits) Always flagged for review

Scoring Rubric (100 points)

Expected Impact: 30 — historical similar experiments

Confidence: 25 — data quality, sample size

Reversibility: 20 — can revert quickly?

Alignment: 15 — matches client goals?

Risk: 10 — downside if wrong?

Guardrails

• One active experiment per campaign (no conflicts)

• Bid changes capped at ±20%

• Circuit breaker: if CPA spikes >30%, pause auto-execute, alert manually

Confidence Evolution

Patterns rated on a three-level scale based on validation across clients:

Low — Observed in 1 client or contradicted once

Medium — Observed in 2 clients, no contradictions OR 1 contradiction + 2 confirmations

High — Observed in ≥3 clients, no contradictions in 30 days. Becomes playbook.
