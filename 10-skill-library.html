The feedback system — for Daragh

Click feedback. Skill gets better.

Every skill card has a Feedback → button. Clicking it opens a pre-filled email that flows through the Gmail ↔ Vault automation. Claude appends the feedback to the skill's feedback.md, adjusts the skill's SKILL.md when patterns emerge, and logs the change.

/ Step 01

Click "Feedback"

Opens a mailto: with subject update_skill_[name] and a structured body. Label auto-applies.

/ Step 02

Daragh fills it in

Rating 1–5, what worked, what didn't, the change they want. Sends it from @ampliodata.io.

/ Step 03

Claude fetches

Gmail MCP picks up the labelled email. Parses the update_skill_ prefix. Routes to the skill folder.

/ Step 04

Vault updates

Feedback appended to feedback.md. If rating ≤2 across 3 inputs, Claude flags the skill for rewrite.

What the feedback form produces

Clicking any Feedback button generates an email that looks like this. Nothing more, nothing less.

To:hello@ampliodata.io
Subject:update_skill_icp-buyer-personas: feedback
Label:Amplio Updates
Skill: icp-buyer-personas
Rating: 4 — good, small improvements needed
What worked: the 5-persona model maps cleanly to our Attio data.
What didn't: the agency-owner outreach sample is too formal for Irish shops.
Change I want: add a second, shorter template variant tuned for small UK/IE agencies.
Route to: 03-Resources/Skills/icp-buyer-personas/feedback.md (append)
01

Feedback never overwrites

Every submission is appended with a timestamp and the sender. The skill's history stays intact.

02

Three signals trigger a rewrite

If a skill gets 3× rating ≤2 in a 30-day window, Claude flags it for rewrite and files a task.

03

Skill-creator handles updates

When a rewrite is needed, the skill-creator skill picks up the feedback and proposes the new SKILL.md.

Who owns what

Ownership of the library.

Every skill has an owner. The owner decides when to rewrite, when to retire, and when to graduate a plugin skill into a core skill.

Daragh

Library curator

Maintains the full skill list. Reviews inbound feedback weekly. Proposes rewrites, deprecations, and new additions. First reviewer on skill-creator outputs.

Walter

Core skills owner

Ships core skills (analytics-tracking, icp-buyer-personas, brand-guidelines-git, attio-crm-enrichment). Reviews the "what changed" diffs.

Plugin authors

Plugin skills

Marketing, Operations, Data, Vault-autoresearch — authored externally. We submit upstream feedback, we don't edit in place.

Claude

Execution + logging

Appends feedback. Flags patterns. Drafts proposed SKILL.md updates when thresholds hit. Never deletes without confirmation.
